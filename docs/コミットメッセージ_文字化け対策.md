# コミットメッセージ文字化け対策

## 問題
PowerShellで `git commit -m "日本語メッセージ"` を実行すると、コミットメッセージが文字化けすることがあります。

## 解決策

### 方法1: 一時ファイルを使用（推奨）
1. UTF-8エンコーディングでコミットメッセージをファイルに保存
2. `git commit -F <ファイル名>` でコミット

```powershell
# コミットメッセージをファイルに書き込む
"修正: 機能追加" | Out-File -FilePath commit_msg.txt -Encoding UTF8

# ファイルからコミットメッセージを読み込む
git commit -F commit_msg.txt

# 一時ファイルを削除
Remove-Item commit_msg.txt
```

### 方法2: Git設定でエンコーディングを指定
```powershell
git config --global i18n.commitencoding utf-8
git config --global i18n.logoutputencoding utf-8
```

### 方法3: コミット前にメッセージを確認
```powershell
# コミット前にメッセージを確認
git log --oneline -1

# 文字化けしている場合は修正
git commit --amend -F commit_msg.txt
```

## 今後の対応
- **コミット前に必ず `git log --oneline -1` でメッセージを確認**
- **文字化けしている場合は、一時ファイルを使用して修正**
- **プッシュ前に再度確認**

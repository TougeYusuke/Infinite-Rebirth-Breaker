# サウンド実装計画

## 概要
Phaser 3を使用したサウンドシステムの実装計画

## 必要なサウンドファイル

### SE（効果音）
1. **攻撃音**
   - `attack_tap.mp3` / `attack_tap.ogg` - タップ攻撃時の音
   - `attack_auto.mp3` / `attack_auto.ogg` - オート攻撃時の音
   - `attack_hit.mp3` / `attack_hit.ogg` - タスクに当たった時の音

2. **タスク関連**
   - `task_spawn.mp3` / `task_spawn.ogg` - タスクが出現した時の音
   - `task_defeat.mp3` / `task_defeat.ogg` - タスクを倒した時の音
   - `task_approach.mp3` / `task_approach.ogg` - タスクが近づいた時の警告音

3. **覚醒モード**
   - `awakening_focus.mp3` / `awakening_focus.ogg` - 集中覚醒発動音
   - `awakening_burst.mp3` / `awakening_burst.ogg` - 爆発覚醒発動音
   - `awakening_creative.mp3` / `awakening_creative.ogg` - 創造覚醒発動音

4. **UI音**
   - `ui_button.mp3` / `ui_button.ogg` - ボタンクリック音
   - `ui_upgrade.mp3` / `ui_upgrade.ogg` - 強化時の音
   - `ui_gameover.mp3` / `ui_gameover.ogg` - ゲームオーバー音

### BGM（背景音楽）
1. **メインBGM**
   - `bgm_main.mp3` / `bgm_main.ogg` - メインゲームのBGM（ループ）
   - `bgm_menu.mp3` / `bgm_menu.ogg` - メニュー画面のBGM（ループ）

## 実装方針

### 1. サウンドマネージャーの作成
- `src/systems/SoundManager.ts` を作成
- Phaser 3の `this.sound` APIを使用
- 音量調整機能（SE/BGM別）
- ミュート機能

### 2. ファイル形式
- **MP3**: 広くサポートされているが、ライセンスに注意
- **OGG**: オープンソース、ブラウザサポートが限定的
- **両方用意**: ブラウザのサポートに応じて自動選択

### 3. パフォーマンス
- 音声ファイルは `public/assets/sounds/` に配置
- 必要に応じてプリロード
- オーディオスプライト（複数のSEを1ファイルにまとめる）も検討

## 実装ステップ

1. **SoundManager.ts の作成**
2. **サウンドファイルの配置場所の決定**
3. **GameScene への統合**
4. **音量調整UIの追加（オプション）**

## 無料音源サイト
- **Freesound.org**: クリエイティブ・コモンズライセンスの音源
- **Zapsplat**: 無料音源（登録必要）
- **効果音ラボ**: 日本語の効果音
- **魔王魂**: ゲーム向けBGM・SE
